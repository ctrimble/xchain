Step 4

The next thing you should know is the different ways a tag can be handled when one skin extends another.

Each page in the nav has a line commented out that looks like the line below.

<!-- <a id="nav-item-1" class="active"/> -->

Go to the index.xhtml, page1.xhtml, and page2.xhtml and uncomment the line in each file.

"Page 1" in the nav now has a special treatment because if you look at the code in base.xhtml we specified that the anchor tags merge. Merging tags will merge content and attributes and new content gets merged to the bottom of the tag.

From base.xhtml
<a id="nav-item-2" href="page1.xhtml" skin:mode="merge">Page 1</a>

The other mode a tag can have is "replace" which will completely replace the tag in the extended document. You've seen this already because all tags by default will replace it's counterpart.
